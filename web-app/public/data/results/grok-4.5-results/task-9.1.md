SSR runs the server on every request, fetching fresh data and sending full HTML; users wait for that work but always see current content—use for personalized or highly dynamic pages. SSG builds HTML at deploy time and serves static files with no per-request server work; users get instant CDN delivery of fixed content—use for docs, marketing, and rarely changing pages. ISR is SSG plus revalidation: the first visitor after the revalidate window may still see the prior page while Next regenerates in the background (or on-demand), then later visitors get the new static HTML—use when content updates on a schedule without full redeploys. Streaming SSR starts the response early, flushing shell HTML while Suspense boundaries stream the rest; the server still runs per request, but users perceive a faster first paint as sections fill in—use for slow data fetches on otherwise server-rendered pages.

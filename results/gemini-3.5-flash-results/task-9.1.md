SSG builds pages once at compile-time, hitting the server only then; users perceive instant loads of static content, ideal for blogs. SSR renders on every request, hitting the server dynamically; users wait for a complete HTML payload but receive real-time data. ISR caches static pages but hits the server in the background to rebuild them after a set interval; users get instant but potentially stale content on first load, seeing updates on reload. Streaming SSR hits the server on every request but pipes HTML chunks to the browser progressively; users perceive a fast initial shell load with interactive components filling in as they load. Choose SSG for static content, SSR for dynamic private pages, ISR for large-scale content sites needing background updates, and streaming SSR for dynamic pages with slow data dependencies.

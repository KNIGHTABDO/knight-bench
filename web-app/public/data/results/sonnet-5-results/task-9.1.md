SSR (App Router default for dynamic routes) renders on the server per request, hitting the server every navigation; the user gets fresh, personalized data but waits for the full round trip before any HTML appears. SSG (generateStaticParams or force-static) renders once at build time; the server is hit only during build, so requests serve prebuilt HTML from a CDN, giving near-instant loads but stale content until redeployed. ISR (revalidate) is static generation with background refresh: the server is hit only when cache expires, regenerating in background, so users mostly get instant cached pages with periodically updated data, never blocking on regeneration. Streaming SSR (Suspense, loading.tsx) hits the server per request like SSR, but sends HTML in chunks as data resolves, so the user sees shell content immediately while slower pieces fill in progressively. Choose SSG for static content, ISR for occasionally-changing content, SSR for personalization, streaming to keep SSR responsive.
